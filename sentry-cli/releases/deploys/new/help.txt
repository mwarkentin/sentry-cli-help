sentry-cli-releases-deploys-new 
Creates a new release deployment.

USAGE:
    sentry-cli releases deploys <VERSION> new --env <ENV>

OPTIONS:
    -e, --env <ENV>                Set the environment for this release.
                                   This argument is required.  Values that make sense here would be
                                   'production' or 'staging'.
        --finished <TIMESTAMP>     Optional unix timestamp when the deployment finished.
    -h, --help                     Prints help information
        --log-level <LOG_LEVEL>    Set the log output verbosity. [possible values: trace, debug,
                                   info, warn, error]
    -n, --name <NAME>              Optional human readable name for this deployment.
    -o, --org <ORG>                The organization slug
    -p, --project <PROJECT>...     The project slug.
        --started <TIMESTAMP>      Optional unix timestamp when the deployment started.
    -t, --time <SECONDS>           Optional deployment duration in seconds.
                                   This can be specified alternatively to `--started` and
                                   `--finished`.
    -u, --url <URL>                Optional URL that points to the deployment.
    -V, --version                  Prints version information
