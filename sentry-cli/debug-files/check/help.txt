sentry-cli-debug-files-check 
Check the debug info file at a given path.

USAGE:
    sentry-cli debug-files check [OPTIONS] <path>

ARGS:
    <path>    The path to the debug info file.

OPTIONS:
        --auth-token <AUTH_TOKEN>    Use the given Sentry auth token.
    -h, --help                       Print help information
        --header <KEY:VALUE>         Custom headers that should be attached to all requests
                                     in key:value format.
        --json                       Format outputs as JSON.
        --log-level <LOG_LEVEL>      Set the log output verbosity. [possible values: trace, debug,
                                     info, warn, error]
        --quiet                      Do not print any output while preserving correct exit code.
                                     This flag is currently implemented only for selected
                                     subcommands. [aliases: silent]
    -t, --type <TYPE>                Explicitly set the type of the debug info file. This should not
                                     be needed as files are auto detected. [possible values: dsym,
                                     elf, pe, pdb, portablepdb, sourcebundle, breakpad, proguard,
                                     wasm]
